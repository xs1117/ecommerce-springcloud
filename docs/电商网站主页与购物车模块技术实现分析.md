## 1. 整体架构设计

### 1.1 项目整体架构

本项目采用 **微服务架构**，基于 Spring Cloud Alibaba 全家桶构建。系统共包含以下微服务模块：

```
┌────────────────────────────────────────────────────────┐
│                    Nacos 注册中心 / 配置中心               │
└────────────────────────────────────────────────────────┘
        │           │           │           │
   ┌────▼────┐ ┌───▼───┐ ┌────▼────┐ ┌───▼────┐
   │Gateway  │ │Dash-  │ │  Cart   │ │ User   │ ...
   │ 网关    │ │board  │ │Service  │ │Service │
   │ :8080  │ │:8081  │ │ :8085   │ │ :8082  │
   └────────┘ └───────┘ └─────────┘ └────────┘
        │
   ┌────▼────┐
   │ Vue3    │
   │ 前端    │
   │ :5173   │
   └─────────┘
```

### 1.2 主页与购物车涉及的服务

| 服务模块 | 端口 | 职责 |
|---------|------|------|
| `ecommerce-frontend` | 5173 | Vue3 单页应用，负责页面渲染与用户交互 |
| `ecommerce-gateway` | 8080 | API 网关，统一入口与路由转发 |
| `ecommerce-dashboard` | 8081 | 主页聚合服务，组装首页所需的各类数据 |
| `ecommerce-cart-service` | 8085 | 购物车微服务，管理与用户绑定的购物车数据 |
| `ecommerce-merchant-service` | 8084 | 商家服务，提供商品/店铺数据 |
| `ecommerce-user-service` | 8082 | 用户服务，提供公告、优惠券等数据 |

### 1.3 数据流向总览

```
用户浏览器 (Vue3)
  │
  ├─ GET /api/home ──────────► ecommerce-dashboard ──► merchant-service (商品/店铺)
  │                            :8081                   user-service (公告)
  │
  ├─ GET /api/home/search ───► ecommerce-dashboard ──► merchant-service (搜索)
  │
  ├─ GET /api/cart/summary ──► ecommerce-cart-service ──► Redis Cluster
  │   POST /api/cart/items      :8085
  │   PATCH /api/cart/items/:id
  │   DELETE /api/cart/items/:id
  │
  └─ WebSocket/CustomEvent ◄── 跨组件通信 (购物车变更通知)
```

#### 3.2.3 语音搜索实现

```javascript
// ── 语音识别核心逻辑 ──
// 基于浏览器原生 Web Speech API (SpeechRecognition / webkitSpeechRecognition)
// 设计思路:
//   1. 检测浏览器是否支持 SpeechRecognition API
//   2. 创建识别实例，设置中文语言(zh-CN)和实时结果(interimResults)
//   3. onresult 回调中过滤非中英文和数字的字符，清洗后填入搜索框
//   4. 错误码映射为用户友好的中文提示:
//      not-allowed      → "请先允许麦克风权限"
//      audio-capture    → "未检测到可用麦克风"
//      network          → "网络异常，语音识别失败"

const toggleSpeechInput = () => {
  const SpeechRecognition = getSpeechRecognitionCtor();
  if (!SpeechRecognition) {
    speechRecognitionSupported.value = false;
    speechTip.value = '当前浏览器不支持语音输入';
    return;
  }
  const recognition = new SpeechRecognition();
  recognition.lang = 'zh-CN';            // 中文识别
  recognition.interimResults = true;      // 返回实时中间结果
  recognition.maxAlternatives = 1;        // 只取最优匹配
  recognition.continuous = false;         // 单次识别即结束

  recognition.onresult = (event) => {
    let transcript = '';
    for (let index = event.resultIndex; index < event.results.length; index += 1) {
      transcript += event.results[index]?.[0]?.transcript || '';
    }
    const clean = sanitizeSpeechKeyword(transcript); // 过滤特殊字符
    if (clean) searchKeyword.value = clean;
  };
  recognition.start();
};
```

```java
/**
 * GET /api/home — 首页数据聚合端点
 *
 * 设计思路（聚合模式）：
 *   Dashboard 服务本身不存储业务数据，它扮演"聚合器"角色：
 *     1. 通过 MerchantCatalogClient 调用 merchant-service 获取热销商品、店铺列表
 *     2. 通过 UserContentClient 调用 user-service 获取商城公告
 *     3. 首页分类、Banner、热搜关键词等静态/配置数据由 Dashboard 直接提供
 *
 * 返回数据结构：
 *   {
 *     hero: { title, subtitle, searchHint },
 *     banners: [{ id, title, subtitle, detail, rules }],
 *     categories: [{ name: "手机数码", icon: "📱" }, ...],
 *     hotKeywords: ["手机", "耳机", ...],
 *     featuredProducts: [{ id, title, price, imageUrls, ... }],
 *     featuredStores: [{ id, title, storeImageUrl, rating, ... }],
 *     notices: [{ id, title, content, detail }],
 *     promotions: [{ id, title, subtitle, detail, rules }]
 *   }
 */
@GetMapping
public ResponseEntity<Map<String, Object>> home() {
    // 1. 获取热销商品（从 merchant-service，取前8条）
    List<Map<String, Object>> featuredProducts = merchantCatalogClient.hotProducts(8)
            .stream().map(this::normalizeProduct).toList();

    // 2. 获取店铺列表（从 merchant-service，取前6条）
    List<Map<String, Object>> featuredStores = merchantCatalogClient.listStores(6)
            .stream().map(this::normalizeStore).toList();

    // 3. 获取公告（从 user-service，如失败则使用默认公告）
    List<Map<String, Object>> notices = userContentClient.activeNotices();
    if (notices == null || notices.isEmpty()) notices = defaultNotices();

    // 4. 组装完整首页数据
    return ResponseEntity.ok(map(
            "hero", map("title", "淘宝风格商城首页", "subtitle", "搜索商品或店铺...",...),
            "banners", defaultBanners(),
            "categories", List.of(map("name","手机数码","icon","📱"), ...),  // 12个类目
            "hotKeywords", List.of("手机","耳机","笔记本","跑鞋","生鲜","美妆"),
            "featuredProducts", featuredProducts,
            "featuredStores", featuredStores,
            "notices", notices,
            "promotions", defaultPromotions()
    ));
}
```

#### 3.3.2 搜索功能实现

```java
/**
 * GET /api/home/search — 多维度搜索
 *
 * 业务规则：
 *   - keyword: 支持商品标题模糊匹配和店铺名称模糊匹配
 *   - type:    "product"仅搜商品 / "store"仅搜店铺 / "all"两者都搜
 *   - sort:    relevance(默认) / price-asc / price-desc / sales-desc / stock-desc / newest
 *   - page/size: 分页从1开始，每页最多48条
 *
 * 实现思路：
 *   1. 一次性从 merchant-service 拉取全部匹配结果（上限500条）
 *   2. 在 Dashboard 内存中完成排序和分页
 *   3. 对于 "all" 模式，将商品和店铺合并后再统一分页
 *   （注：这个方案适合数据量较小的学习项目，生产应改为后端分页）
 */
@GetMapping("/search")
public ResponseEntity<Map<String, Object>> search(
        @RequestParam("keyword") String keyword,
        @RequestParam(value = "type", defaultValue = "all") String type,
        @RequestParam(value = "sort", defaultValue = "relevance") String sort,
        @RequestParam(value = "page", defaultValue = "1") int page,
        @RequestParam(value = "size", defaultValue = "24") int size) {

    // 拉取全部匹配数据
    List<Map<String, Object>> allProducts = merchantCatalogClient
            .searchProducts(normalizedKeyword, normalizedSort, 500)
            .stream().map(this::normalizeProduct).toList();

            .searchStores(normalizedKeyword, normalizedSort, 500)

    List<Map<String, Object>> items = switch (normalizedType) {
```

#### 3.3.3 个性化推荐

```java
/**
 * GET /api/home/recommendations — 基于搜索历史的商品推荐
 *
 * 实现思路（简化版）：
 *   1. 接收前端的搜索历史（格式: "product:手机,product:耳机"）
 *   2. 解析历史记录，提取关键词
 *   3. 将关键词作为参数传给 merchant-service 的推荐接口
 *   4. merchant-service 根据关键词匹配相关商品
 *
 * 参数说明：
 *   history: 搜索历史，格式 "type:keyword,type:keyword"
 *   limit:   返回推荐数量上限（默认8，最大12）
 */
@GetMapping("/recommendations")
public ResponseEntity<Map<String, Object>> recommendations(
        @RequestParam(value = "history", required = false) String history,
        @RequestParam(value = "limit", defaultValue = "8") int limit) {
    List<String> keywords = parseHistory(history);
    List<Map<String, Object>> items = merchantCatalogClient
            .recommendProducts(keywords, Math.max(1, Math.min(limit, 12)))
            .stream().map(this::normalizeProduct).toList();
    return ResponseEntity.ok(map("keywords", keywords, "items", items));
}
```

#### 4.2.1 全局悬浮按钮 — CartFloatingButton.vue

**文件路径**：[CartFloatingButton.vue](file:///d:/桌面/项目/软工实训/ecommerce-frontend/src/components/CartFloatingButton.vue)

```javascript
// ── 核心机制 ──
// 1. 组件挂载时同时请求 count（商品总数）和 summary（包含选中金额）
// 2. 显示逻辑: 有 token 才显示（登录后才能看到购物车按钮）
// 3. 实时更新: 监听两个事件源
//    - ecommerce:cart-changed 自定义事件（由 cart.js 触发）
//    - window storage 事件（跨标签页同步）

const load = async () => {
  if (!token.value) { count.value = 0; amount.value = 0; return; }
  const [nextCount, summary] = await Promise.all([
    fetchCartCount(),      // GET /api/cart/count → { count: N }
    fetchCartSummary()     // GET /api/cart/summary → { items, selectedAmount, ... }
  ]);
  count.value = nextCount;
  amount.value = Number(summary?.selectedAmount ?? summary?.totalAmount ?? 0);
};

onMounted(() => {
  load();
  offChanged = onCartChanged(load);        // 监听同一页面内的购物车变更
  window.addEventListener('storage', load); // 监听跨标签页的 storage 变更
});
```

```java
 *
 * 基础路径: /api/cart

     *       + totalAmount + selectedAmount + behaviors

    @PostMapping("/items")

    private AuthenticatedUser currentUser(Authentication authentication) {
```

```java
/**

 * 1. 购物车商品 — Hash 结构
```

##### B. 添加商品 — addItem() 方法

```java
/**
 * 添加商品到购物车的完整流程:
 *
 * Step 1: 参数校验
 *   - productId 不能为空
 *   - title 不能为空
 *   - price 不能为空且 >= 0
 *
 * Step 2: 检查是否已存在
 *   - 按 productId 在已有商品中查找
 *   - 如果已存在: 执行数量累加逻辑
 *   - 如果不存在: 创建新购物车项
 *
 * Step 3: 获取商品库存（跨服务调用 merchant-service）
 *   - 调用 GET /api/merchant/internal/products/{id}/availability
 *   - 获取 stock(库存) 和 sellable(是否可售)
 *   - 商品被平台下架(ADMIN_BLOCKED) → 返回 "已被平台下架，暂不可购买"
 *   - 库存为0 → 返回 "库存不足，暂不可加入购物车"
 *
 * Step 4: 数量上限计算
 *   maxQuantity = min(当前库存, 用户传入的maxQuantity, 99)
 *   保障三层限制: 库存上限 > 用户设定上限 > 系统硬上限
 *
 * Step 5: 数据持久化
 *   - 已有商品: 叠加数量 + 更新 maxQuantity(取两者较小值)
 *   - 新商品:   生成 UUID 作为 itemId, 记录 createdAt
 *   - 统一:     更新 updatedAt, 写入 Redis Hash
 *
 * Step 6: 记录行为
 *   - action = "ADD_ITEM"
 *   - LPUSH 到行为列表, LTRIM 保持100条
 */
public Map<String, Object> addItem(Long userId, AddCartItemRequest request) {
    validateAddRequest(request);

    // 加载用户已有商品，检查是否重复
    List<Map<String, Object>> items = loadItems(userId);
    Map<String, Object> existing = findItemByProductId(items, request.productId())
            .orElse(null);

    // 跨服务获取商品库存与可购状态
    ProductAvailability availability = fetchProductAvailability(request.productId());
    ensureSellable(availability);

    // 三级上限: 库存 > 用户设定 > 系统硬上限(99)
    int maxQuantity = Math.min(availability.stock(),
            resolveMaxQuantity(existing != null
                    ? intValue(existing.get("maxQuantity"), ...)
                    : request.maxQuantity()));

    // 已有商品: 累加数量; 新商品: 创建记录
    int nextQuantity = existing == null
            ? quantity
            : intValue(existing.get("quantity"), 1) + quantity;

    if (nextQuantity > maxQuantity) {
        throw new ResponseStatusException(HttpStatus.BAD_REQUEST, "数量不能超过可购上限");
    }

    // ...构造 saved Map, 写入 Redis, 记录行为...
    upsertItem(userId, saved);
    recordBehavior(userId, toBehaviorMap("ADD_ITEM", ...));
    return normalizeItem(saved);
}
```

##### C. 更新商品 — updateItem()

```java
/**
 * 更新购物车商品（数量/选中状态/来源信息）
 *
 * 与 addItem 的关键区别:
 *   - 按 itemId 精准定位（而非 productId）
 *   - quantity 直接替换而不累加
 *   - 每次更新都会重新验证库存，确保数量不超过最新库存
 */
public Map<String, Object> updateItem(Long userId, String itemId,
                                       UpdateCartItemRequest request) {
    Map<String, Object> item = getItemMap(userId, itemId);

    if (request.quantity() != null) {
        // 更新数量前重新拉取库存，防止超买
        ProductAvailability availability = fetchProductAvailability(
                longValue(item.get("productId"), null));
        ensureSellable(availability);
        int maxQuantity = Math.min(
                intValue(item.get("maxQuantity"), 99),
                availability.stock());
        if (request.quantity() > maxQuantity) {
            throw new ResponseStatusException(HttpStatus.BAD_REQUEST, "数量不能超过可购上限");
        }
        item.put("quantity", clampQuantity(request.quantity()));
        item.put("maxQuantity", maxQuantity);
    }

    // selected 为 null 时保持原值不变（支持只更新数量不改变选中状态）
    if (request.selected() != null) item.put("selected", request.selected());

    upsertItem(userId, item);
    recordBehavior(userId, toBehaviorMap("UPDATE_ITEM", ...));
    return normalizeItem(item);
}
```

##### D. 汇总计算 — buildSummary()

```java
/**
 * 构建购物车汇总数据
 *
 * 统计维度:
 *   - totalQuantity:     全部商品总件数
 *   - selectedQuantity:  已勾选商品总件数
 *   - totalAmount:       全部商品总金额（BigDecimal精确计算）
 *   - selectedAmount:    已勾选商品总金额
 *   - lineAmount:        每个商品的"单价×数量"小计
 *
 * 金额处理规则:
 *   - 使用 BigDecimal 避免浮点数精度问题
 *   - normalizeMoney(): 统一保留两位小数，HALF_UP 四舍五入
 */
private Map<String, Object> buildSummary(Long userId,
                                          List<Map<String, Object>> items) {
    int totalQuantity = 0, selectedQuantity = 0;
    BigDecimal totalAmount = BigDecimal.ZERO, selectedAmount = BigDecimal.ZERO;

    for (Map<String, Object> item : items) {
        int quantity = intValue(item.get("quantity"), 1);
        BigDecimal price = moneyValue(item.get("price"));
        BigDecimal lineAmount = price.multiply(BigDecimal.valueOf(quantity));

        totalQuantity += quantity;
        totalAmount = totalAmount.add(lineAmount);
        item.put("lineAmount", normalizeMoney(lineAmount));  // 每行小计

        if (Boolean.TRUE.equals(item.get("selected"))) {
            selectedQuantity += quantity;
            selectedAmount = selectedAmount.add(lineAmount);
        }
    }

    result.put("totalAmount", normalizeMoney(totalAmount));
    result.put("selectedAmount", normalizeMoney(selectedAmount));
    result.put("behaviors", recentBehaviors(userId, 10));
    return result;
}
```

##### E. 行为记录系统

```java
/**
 * 用户购物车行为追踪系统
 *
 * 记录的操作类型(action):
 *   ADD_ITEM      — 添加商品
 *   UPDATE_ITEM   — 更新数量/选中状态
 *   REMOVE_ITEM   — 删除单个商品
 *   CLEAR_CART    — 清空购物车
 *
 * 存储策略:
 *   - 使用 Redis List (左进左出队列)
 *   - LPUSH 写入最新记录到队首
 *   - LTRIM 裁剪到100条，自动淘汰旧记录
 *   - 查询时取 limit 条（默认10条），从队首开始
 *
 * 容错设计:
 *   - recordBehavior 的异常被 catch 忽略（行为记录失败不影响核心业务）
 *   - parseMap 失败时抛出 500（数据一致性问题必须暴露）
 */
private void recordBehavior(Long userId, Map<String, Object> event) {
    try {
        String key = cartBehaviorKey(userId);
        redisTemplate.opsForList().leftPush(key, objectMapper.writeValueAsString(event));
        redisTemplate.opsForList().trim(key, 0, BEHAVIOR_LIMIT - 1);
    } catch (Exception ignored) {
        // 行为记录失败不影响核心功能，静默忽略
    }
}
```

#### 4.4.3 安全认证机制

```java
/**
 * ── 购物车服务的安全设计 ──
 *
 * 认证链路:
 *   1. 前端 Axios 拦截器自动在每个请求头注入 Bearer Token
 *   2. TokenAuthenticationFilter 拦截所有请求，解析 Authorization Header
 *   3. 通过 TokenService.parseToken() 验证签名并还原 AuthenticatedUser
 *   4. 将 AuthenticatedUser 设为 SecurityContextHolder 的 Principal
 *   5. CartController 通过 Authentication.getPrincipal() 获取用户信息
 *
 * 安全配置要点:
 *   - 关闭 CSRF (前后端分离 + Token 认证，不存在 CSRF 风险)
 *   - STATELESS 会话 (不创建服务端 Session，每次请求独立认证)
 *   - 所有 /api/cart/** 需要认证 (anyRequest().authenticated())
 *   - /actuator/** 放行 (健康检查端点无需认证)
 */
@Configuration
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http,
                                                    TokenAuthenticationFilter filter)
            throws Exception {
        http.csrf(csrf -> csrf.disable())
                .cors(Customizer.withDefaults())
                .sessionManagement(session ->
                        session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers("/actuator/**").permitAll()
                        .anyRequest().authenticated())
                .addFilterBefore(filter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }
}
```

#### 4.4.4 服务发现 — IntegrationEndpointResolver

```java
/**
 * 集成端点解析器
 *
 * 解决核心问题: 在微服务环境中，cart-service 如何找到 merchant-service 的地址？
 *
 * 策略（两层回退）:
 *   1. 优先通过 Nacos 服务发现: discoveryClient.getInstances("ecommerce-merchant-service")
 *      → 获取动态注册的服务实例列表 → 取第一个实例的 URI
 *   2. 如果 Nacos 不可用（本地开发/测试），回退到配置文件中的 base-url:
 *      app.integration.merchant.base-url = http://localhost:8084
 *
 * URL 规范化:
 *   - 确保以 http:// 或 https:// 开头
 *   - 去掉尾部斜杠，避免拼接出双斜杠
 */
@Component
public class IntegrationEndpointResolver {

    public String resolveBaseUrl(String serviceId, String fallbackBaseUrl) {
        // 1. 尝试 Nacos 服务发现
        if (StringUtils.hasText(serviceId)) {
            List<ServiceInstance> instances = discoveryClient.getInstances(serviceId.trim());
            if (instances != null && !instances.isEmpty()) {
                return normalizeBaseUrl(instances.getFirst().getUri().toString());
            }
        }
        // 2. 回退到静态配置
        if (StringUtils.hasText(fallbackBaseUrl)) {
            return normalizeBaseUrl(fallbackBaseUrl);
        }
        throw new IllegalStateException("No available target endpoint");
    }
}
```

```
        Hash: cart:items:{uid}   List: cart:behavior:{uid}
        (购物车商品数据)           (用户操作行为记录)
```

### 6.3 模块交互关系总结

```
                         ┌───────────────────┐
                         │   Nacos 注册中心    │
                         │  (服务发现 + 配置)   │
                         └──────┬────────────┘
                                │ 注册/发现
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
   ┌────▼────┐            ┌────▼────┐            ┌─────▼─────┐
   │Dashboard│            │  Cart   │            │ Merchant  │
   │ :8081   │◄───────────│ Service │───────────►│ Service   │
   │ 聚合层  │  不直接交互  │ :8085   │ 查库存/商品 │  :8084    │
   └────┬────┘            └────┬────┘            └───────────┘
        │                      │
        │ GET /api/home        │ GET|POST|PATCH|DELETE
        │ GET /api/home/search │ /api/cart/*
        │                      │
   ┌────▼──────────────────────▼────┐
   │          Vue3 前端 SPA         │
   │ 局部状态: ref() + localStorage │
   │ 跨组件通信: CustomEvent        │
   │ API 请求: Axios + Token 拦截   │
   └────────────────────────────────┘
```

**关键交互说明**：

| 交互方向 | 说明 |
|---------|------|
| 前端 → Dashboard | Dashboard 是首页数据的聚合入口，前端不直接调 merchant/user |
| 前端 → Cart Service | 购物车直接由前端调用，无需经过 Dashboard |
| Cart → Merchant | 购物车服务需要实时验证库存，通过 Nacos 发现 merchant 地址 |
| Dashboard → Merchant | Dashboard 通过 DiscoveryClient 发现 merchant 地址后调用 |
| Dashboard → User | Dashboard 调用 user-service 获取公告 |
