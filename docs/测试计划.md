## 3. 测试目标

- 验证系统核心业务链路的可用性，包括登录认证、首页聚合、商品浏览、购物车、订单、支付、聊天和 AI 导购。
- 验证前端工程可构建、可访问，核心页面路由与未登录拦截逻辑符合设计预期。
- 识别当前实验环境中的阻塞项，区分环境问题与业务缺陷，避免对课程验收形成误判。

## 4. 测试范围

### 4.1 包含范围

- 前端：登录页、首页、搜索页、商品详情、店铺详情、购物车、结算、订单、商家中心、管理员审核、聊天页。
- 网关与微服务：`gateway`、`dashboard`、`user`、`merchant`、`cart`、`inventory`、`order`、`payment`、`chat`、`ai`。
- 基础链路：用户认证、首页聚合查询、购物车与订单接口、AI 问答入口。
- 非功能：前端构建验证、基础可访问性、轻量性能压测。

### 4.2 不包含范围

- 外部大模型效果准确率评估。
- 长稳压测、容量评估、容灾与安全渗透测试。
- 生产环境兼容性和多终端适配的全量覆盖。

## 7. 测试策略

### 7.1 功能测试

- 采用黑盒测试方法，按“用户登录 -> 首页浏览 -> 商品检索 -> 交易入口 -> AI 入口”主链路验证。
- 对页面功能采用 Playwright 截图取证和人工结果核验结合方式。
- 对接口功能采用 HTTP 请求验证状态码与响应行为。

### 7.2 性能测试

- 使用 `autocannon` 对前端静态预览页和网关首页接口进行 5 秒轻量压测。
- 记录平均响应时间、吞吐量、非 2xx 比例，作为实验项目的基础性能参考值。

### 7.3 兼容性与可用性测试

- 以 Chromium 内核浏览器为最小可用基线。
- 验证未登录访问受保护路由时是否回落到登录页。

### 7.4 缺陷判定原则

- 接口返回 5xx、核心服务未监听、网关不可用，判定为阻塞级问题。
- 前端可构建但依赖后端接口失败，前端构建判定通过，联调链路判定不通过。
- 仅有历史日志、没有本次实测证据的内容，不在结果报告中判定为通过。

## 10. 风险与预案

| 风险 | 影响 | 预案 |
|---|---|---|
| Maven 未安装 | 无法重新构建后端 Jar 或启动缺失服务 | 基于现有 `target/classes`、日志和实测结果完成复查 |
| 核心服务端口未监听 | 网关联调结果失真或全部失败 | 将“服务未启动/端口被占用”单独归类为阻塞项 |
| 外部依赖未就绪 | 购物车、订单、支付、聊天等链路无法完整执行 | 仅输出已验证结果，不扩展推断 |
| AI 服务依赖外部模型或向量库 | AI 问答无法稳定复现 | 仅验证入口存在性和配置依赖，不输出效果性结论 |

## 12. 通过准则

- 前端构建成功，登录页在浏览器可正常渲染。
- 核心 API 至少满足首页查询和用户登录两条基础链路返回 2xx。
- 如接口链路存在 5xx 或服务未监听，则版本整体不判定为“测试通过”。
